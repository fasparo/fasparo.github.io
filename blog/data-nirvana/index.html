Data nirvana

2015-12-17

A personal computer is firstly a tool, but it was also intended to be place to store personal data. In 2015 most people have given up and entrusted all this stuff to corporate clouds. But I have doubled down and decided that my personal data (or at least the canonical version of it) belongs in a personal place, not in a data center. And after a decade or so on Linux, I have arrived at what feels like the ideal way to do that.

First, definitions. Personal data seems to fall into two categories: * archive data – meaning all those documents, pictures, videos, correspondence, perhaps audio, perhaps computer code, which you do not want to lose, ever; and * current data – the things you are working on right now.

Next, requirements:

  • current data must have 2 copies, for protection against technical failure (the most likely catastrophe);

  • archive data must have 3 copies, including a remote one for protection against fire and theft;

  • all data should be as separate as possible from the operating system – so that changing or reinstalling the OS does not require a weekend of preparation.

My solution, then, is to:

  • keep current data in /Desktop and archive data on an always-connected external drive labeled Data;

  • symlink /Desktop back into Data (and any important config files back into /Desktop;

  • do a weekly rsync of Data to a second external drive labeled Backup, with the -L flag so that the content of /Desktop is pulled along too; and

  • do a less frequent rsync of Data to a remote drive, without the -L flag so that only archive data is transferred.

Requirements met. Data safe, privacy protected, and all while respecting a valuable founding principle of personal computing.
