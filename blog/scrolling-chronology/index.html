Chronological scrolling

2014-03-28

As a vertical medium, the web has a unresolved problem with chronology. How should discrete units of news content be organized? Blogging introduced the obvious convention of antichronological order: latest on top. Other streams followed: social feeds, RSS, more than a few article comment sections (although not usually those on blogs), and of course the private news feed of webmail.

But reading, like writing, is done from top to bottom. Antichronological order favors the latest, but chronological order favors the story – and news is by definition a story. Just as a chat thread is not shown with the latest pings on top, there is something illogical about reading articles or comments or tweets in antichronological order. Email, being a list of action items, highlights an additional downside: dealing with the latest item first is an inefficient way to Get Things Done.

In reality, antichronological order is only ideal when there is both (1) no need to see everything, and (2) no sequential aspect to the content. Some would argue that is the case with social media news. But in practice there is surely a lot of the following going on:

  • downscrolling (or downswiping) and reading items in wrong order
  • locating the last read item and upscrolling to read the latest

In fact, the major problems with chronological order are technical:

  • a chronological news feed needs a defined start point, which must be tracked per-user
  • you cannot scroll up beyond the top of the screen

In 2014 such problems are easily solved. Here's a solution:

  • A news feed tracks items *seen* (not necessarily "read") and shows the last unseen item first by default, but with an alternative "straight to latest" stick-to-bottom mode, like in chat. A blog can show the last item only – in the age of personalized feeds we can safely assume that homepage visitors are mainly newcomers.

  • Ajax-based upscroll functionality allows the user to backtrack over previous items. If Javascript is unavailable then the view falls back to antichronological order, allowing access by downscrolling as in the illogical tradition.

A useful convention would thus become more useful still.
