Defining the web

2013-09-25

Like all responsible citizens, I back up my files both locally and online. My online backup service has a web interface, which is occasionally useful but also uncomfortable – after all, it means my private files are technically "on the web".

Or does it? How about webmail? Those who dismiss Facebook's privacy settings, insisting that "you must assume it's public if you put it online", presumably don't think the same of their email. But it too has a URL.

In reality, "on the web" is a concept of graded reality:

  1. it has a URL starting with "http" – you access it using the web's protocol, HTTP;
  2. it has a unique URL starting with "http" – only a URL pointing to a single document is useful;
  3. it has a unique URL starting with "http" and no authentication – at this point all someone needs is the link, hence...
  4. it has a unique URL starting with "http", no authentication, and is linked to a page with all the same characteristics

I submit that only web pages in the category 4 are unambiguously "on the web". In category 3, search results pages are a classic border case. As for the shady category 2, the term "dark web" is starting to be useful. And the list of my files at GenericWebBackup.com, with a shared URL for everyone and armor-plated password security, is really not on the web at all. That's a relief.
